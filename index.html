Handycart

Subscription-based e-commerce web portal

For this school project, we had to develop a web solution implementing UI design standards. Care had to be taken while designing this website in order to achieve the right balance of colors & white space. Famous rules such as the golden ratio and rule of thirds were also implemented.

The design was first established by creating user personas and design personas. Based on these, a theme and personality was given to the site in the form of a mascot, typography and logo.

When a user subscribes to a product, they can choose the days, quantities and seller from which they would like to receive their goods. On a recurring basis ( weekly, monthly, bi-weekly), these products are delivered to the user without the hassle of manual effort in ordering them again.

Technologies used: Python, Flask, HTML5, CSS (Bootstrap), jQuery, SQLite, Heroku

Penetration Testing

Perform analysis and testing to prevent security breaches

Tennis prediction system

MongoDB-based tennis result predictor

Using 10-15 years worth of tennis statistical data from various tennis websites and betting sites, a table was created displaying information for players in a given match or head-to-head.

The data was stored in MongoDB as collections. Based on this data, a player's rank, win-ratio, career wins, career losses and power rating were calculated and shown. With precise information on each match result, a user visiting this site has a fair idea of which player has an edge in their rivalry. A graphical representation of betting trends at a given point (or a critical point) of the match gives the user a sense of where the game is leading to.

As mentioned above, MongoDB was used to handle the volume of data from betting sites. HTML, CSS and jQuery was used for the front-end and Java was used for the back-end with heavy emphasis on JSON.

Climalite

Weather application powered by Node & Express

This application was built using Express, the popular web framework based on Node

Axios, a well-known HTTP client in the npm community was utilized in fetching the data and serving the API calls. For the latter, Google Geolocation API was used to fetch latitude, longitude and other geographical details for the searched city.

Successful Promise chains would redirect the values to the Dark Sky API which fetches the weather details for the given geographical data.

While this project may seem simple, it helped me reinforce my understanding of Promises and effective use of Express and API's

Tools & Technologies used: Node, Express, Axios, Handlebars, Visual Studio Code
